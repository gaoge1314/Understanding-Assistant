# 转录本处理器

## 职责

将非教材类的学习材料（课堂录音文字版、课堂笔记）预处理为结构化文本，供下游的知识点提取和卡片生成使用。

**关键原则**：
- 保留老师原话（用「」标记关键表述）
- 纠正识别不准确的地方
- 不丢失信息，只重组不删减

## 处理流程

### 第一步：类型识别

根据内容特征判断输入类型：

| 特征 | 判定 | 处理策略 |
|:---|:---|:---|
| 口语化强（嗯、啊、那个）、有填充词、按时间顺序记录 | 课堂录音文字版 | 完整清洗+主题重组 |
| 碎片化、简写、关键词罗列、无完整句子 | 课堂笔记 | 主题聚合+补全 |
| 结构完整、章节清晰、术语正式 | 教材/正式文档 | 跳过（不需要此步骤） |

### 第二步：逐文档独立处理

对每个输入文件单独执行：

#### ① 清洗层（仅录音稿适用）

- **去填充词**：去除口语填充词（嗯、啊、呃、就是说、然后呢、那个、这个）
- **合并重复**：同一观点重复多遍 → 保留最完整的一遍，其余删除
- **保留原话**：老师的关键定义、结论、精辟表述用 `「」` 引号标记
- **纠正错误**：自动修正明显的识别/转录错误（如"火三喷发"→"火山喷发"）

#### ② 主题拆分层

- 扫描全文，识别话题转换的信号词（"接下来我们讲……"、"换一个话题……"）
- 每个独立话题切分为一个段落
- 为每个段落标注主题关键词

#### ③ 重组层

- 将时间顺序转换为主题顺序
- 同一主题下的内容按逻辑排序：概念 → 原理 → 案例 → 结论
- 保留原始时间顺序标记作为辅助信息（可选）

#### ④ 输出

```
输出文件：{原始名称}_已处理.md
保存位置：output/processed/{原始名称}_已处理.md
```

### 第三步：跨文档统一整合

当有多个文档需要合并时执行：

#### ① 合并层

- 识别跨文档的重复/重叠内容
- 合并策略：
  - 教材内容作为结构骨架
  - 老师原话优先保留（`「」`标记）
  - 笔记中的补充说明附加在对应章节下

#### ② 章节排序层

- 有教材时：按教材章节顺序排列
- 无教材时：按逻辑顺序（基础知识→进阶知识→应用）
- 生成目录

#### ③ 输出

```
输出文件：{主题}_已整合.md
保存位置：output/processed/{主题}_已整合.md
```

## 输出格式规范

### 已处理.md 格式（逐文档）

```markdown
# {原始名称} - 已处理

> 来源类型：[课堂录音文字版 | 课堂笔记]
> 处理方式：清洗+主题拆分+重组

## {主题1}

`「老师原话」`：（此处用引号保留老师的原始表述）

[整理后的内容……]

## {主题2}

……
```

### 已整合.md 格式（跨文档合并）

```markdown
# {科目名称} - 已整合学习材料

> 来源：课堂录音（第一讲、第二讲）、课堂笔记、教材
> 处理方式：逐文档预处理 + 跨文档整合

## 第一章：{主题名称}

### 1.1 {子主题}

**老师原话**：「……」
**教材对应**：[教材内容概括]
**补充**：[笔记/录音中的展开说明]

### 1.2 {子主题}

……

```

## 质量检查

- [ ] 口语填充词已去除
- [ ] 老师原话已用「」标记
- [ ] 明显错误已纠正
- [ ] 已按主题重组（非时间顺序）
- [ ] 跨文档内容已去重合并
- [ ] 章节顺序合理
- [ ] 输出为 Markdown 格式
