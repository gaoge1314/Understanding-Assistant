> 日期：2026-06-03
> 状态：设计稿（待审批）

## 一、概述

### 1.1 动机

基于用户对 v3 学习应用的体验反馈，进行 5 项交互增强：知识接口可点击跳转、今日计划选卡系统、用户笔记增强（可替换答案）、背诵提示功能、默认全部展开。

### 1.2 设计目标

- 知识接口从纯文本升级为可交互的跳转链接
- 新增"今日计划"入口，支持双模式选卡（按组选择 / 智能推荐）
- 用户笔记增强为"创作模式"，支持替代原始答案
- 背诵视图增加"提示"按钮，展示关键词线索
- 所有折叠面板默认展开
- 保持纯前端、零依赖、模板化架构不变

### 1.3 本次范围

- **做**：知识接口链接可点击跳转
- **做**：今日计划按钮 + 双模式选卡模态框 + 自动创建背诵组
- **做**：用户笔记增强（创作编辑区 + replaceAnswer 开关）
- **做**：背诵提示（关键词展示 + 可编辑 + 数量设置）
- **做**：默认全部展开
- **做**：关键词可编辑
- **不做**：移动端适配、PWA、内嵌 AI 追问、检测题功能

## 二、术语对照

| 术语 | 说明 |
|:---|:---|
| **今日计划** | 每日背诵前的选卡入口，双模式选卡，自动创建背诵组 |
| **背诵组** | 今日计划创建的临时组（"第一组""第二组"...），不显示在侧边栏 |
| **默认组** | 存放未分入自定义组的卡片，始终存在，位于侧边栏底部 |
| **提示** | 背诵模式下展示的关键词摘要，取自 memory_techniques.keywords |
| **用户笔记（增强）** | 原用户笔记 + replaceAnswer 开关，开启后可替代原始答案 |

## 三、功能详细设计

### 3.1 知识接口链接可点击

**位置**：app.js → `formatInterfaces()` 函数

**解析规则**：
- 知识接口文本格式：`→ 标题：说明` 或 `→ 标题`
- 提取冒号前的文本作为目标卡片标题
- 在 `cardData` 中按标题精确匹配

**渲染**：
```html
<a class="knowledge-link" data-card-index="3">→ 灾害系统动力学机制：系统各要素的动态关系</a>
```

**点击行为**：`.knowledge-link` 点击 → 获取 `dataset.cardIndex` → 调用 `selectCard(index)`

**CSS**：蓝色下划线样式，hover 加深 + 背景色

**边界情况**：
- 匹配不到目标卡片 → 渲染为纯文本（降级）
- 同一张卡引用自身 → 点击不跳转（或滚动到顶部）

---

### 3.2 今日计划

#### 3.2.1 入口

顶部导航新增"📋 今日计划"按钮，位于 Tab 导航右侧、"↺"恢复按钮左侧。

#### 3.2.2 模态框 — 双模式选择

模态框结构：
```
┌─────────────────────────────────────────┐
│  📋 今日计划                            │
│                                         │
│  [按组选择]  [智能推荐]                  │  ← Tab 切换
│                                         │
│  ┌─── Tab1: 按组选择 ──────────────┐   │
│  │  ☑ 基础概念 (3张, 120字)        │   │
│  │  ☐ 灾害管理 (2张, 80字)         │   │
│  │  ☑ 默认组 (5张, 200字)          │   │
│  │                                 │   │
│  │  已选: 2组, 8张, 320字          │   │
│  └─────────────────────────────────┘   │
│                                         │
│  ┌─── Tab2: 智能推荐 ──────────────┐   │
│  │  推荐 ${N} 张 (目标: 500字)      │   │
│  │                                 │   │
│  │  ☑ 灾害系统      Level 3 待复习  │   │  ← 每项可
│  │  ☑ 灾害链        新卡片         │   │  ← 手动勾选
│  │  ☐ 山地灾害      新卡片         │   │
│  │  ...                            │   │
│  │                                 │   │
│  │  已选: 4/8 张, 180字            │   │
│  └─────────────────────────────────┘   │
│                                         │
│  上一组"第一组"还有 2 张未完成           │  ← 如果有未完成
│  ☑ 将未完成卡片纳入新组                  │  ← 勾选框
│                                         │
│  [取消]  [开始背诵]                      │
└─────────────────────────────────────────┘
```

**Tab1 - 按组选择**：
- 列出学习视图的所有自定义组 + 默认组
- 显示每组名称、卡片数、总字数
- 点击组名勾选/取消
- 底部汇总已选组数、卡片数、字数

**Tab2 - 智能推荐**：
- 沿用 `computeRecommendation()` 算法
- 以卡片为单位展示，每项可单独勾选/取消
- 显示卡片熟悉状态（Level / 待复习 / 新卡片）
- 底部汇总已选数/推荐数/字数

#### 3.2.3 自动创建背诵组

确认后行为：
1. 未完成的上组卡片（如勾选纳入）合并到新组
2. 命名"第 N 组"，N = 已有"第X组"数量 + 1
3. 数据存入 `appState.recitationGroups`（与 `appState.groups` 分离）
4. 自动跳转到背诵视图，以该组为范围

**数据结构**：
```javascript
recitationGroups: [
  {
    name: "第一组",
    cardIds: [0, 2, 5, 8],      // 卡片索引
    created: "2026-06-03",
    completed: false,
    currentIndex: 0               // 背诵进度
  }
]
```

#### 3.2.4 背诵视图联动

- 进入背诵视图时检查 `recitationGroups`
- 如果有活跃组，以此组范围背诵
- 组内按 `cardOrder` 顺序
- 每张卡片的"记得/忘了"结果更新到 `reviewRecord`（全局，不限于组）
- 组内所有卡片完成后标记该组 `completed: true`
- 显示完成统计

### 3.3 用户笔记增强（创作模式）

#### 3.3.1 布局

在卡片折叠面板中新增"⑤ 我的创作"，位置在"④ 知识接口"之后：

```
┌─ ⑤ 我的创作 ─────────────────────┐  ← 默认展开
│  ┌─────────────────────────────┐  │
│  │ (多行编辑区)                 │  │
│  │ 在此写下自己的理解或扩展创作  │  │
│  └─────────────────────────────┘  │
│  [保存]  [清空]                    │
│                                    │
│  ◉ 用自己的创作替代原始答案         │  ← toggle switch
│  ○ 仅作为笔记，不替代答案           │
│                                    │
│  ─── 已保存的预览 ───              │
│  (用户创作内容原文展示)              │
└────────────────────────────────────┘
```

#### 3.3.2 数据存储

在 `cardStates[title]` 中扩展：
```javascript
cardStates[title] = {
  folded: { ... },
  myUnderstanding: '',      // 原有—右侧栏笔记
  myCreation: '',            // ★ 新增—创作内容
  replaceAnswer: false,      // ★ 新增—替代开关
  isCompleted: false
}
```

#### 3.3.3 替代逻辑

- `replaceAnswer === true` 且 `myCreation` 非空时：
  - 主内容区的"① 答案"渲染创作内容
  - 背诵视图翻转卡背面显示创作内容
  - 所有展示原始答案的地方都替换
- `replaceAnswer === false` 或 `myCreation` 为空时：
  - 显示原始答案，和之前一样

#### 3.3.4 格式支持

支持简单的行内 Markdown 渲染：
- `**粗体**` → `<strong>`
- `*斜体*` → `<em>`
- 换行 → `<br>`
- `- 列表` → `<li>`

### 3.4 背诵提示功能

#### 3.4.1 翻转卡正面

```
┌──────────────────────────────┐
│           灾害系统            │
│                              │
│        [💡 显示提示]          │  ← 蓝色按钮
│                              │
│  ── 提示内容 ──              │  ← 点击后显示
│  · 灾害系统四要素             │
│  · 孕灾环境→致灾因子→...     │
│                              │
│    👆 点击翻转查看答案        │
└──────────────────────────────┘
```

#### 3.4.2 提示内容来源

- 从 `memory_techniques.keywords` 取前 N 条
- N 为 `userSettings.hintCount`（默认 2），用户在设置中调整
- 每条截取前 25 个字符展示
- 用户可在学习视图"③ 记忆技巧"中编辑关键词，编辑后自动保存到 `cardStates`

#### 3.4.3 交互

- 初始状态：仅显示"💡 显示提示"按钮，提示内容隐藏
- 点击按钮 → 显示提示内容，按钮文字变为"🙈 隐藏提示"
- 再点击 → 隐藏提示
- 提示的显示/隐藏状态不影响翻转功能
- 翻转卡片后提示自动隐藏

#### 3.4.4 关键词编辑

在"③ 记忆技巧"板块中：
- 关键词列表每条显示为可编辑文本框
- 用户修改后自动保存（blur 或按 Enter）
- 修改后的关键词存储在 `cardStates[title].editedKeywords`
- 任何时候清空编辑，恢复原始关键词

### 3.5 默认全部展开

#### 3.5.1 折叠面板

```javascript
// 原来
folded: { answer: true, path: true, memory: false, interface: false }
// 改为
folded: { answer: true, path: true, memory: true, interface: true }
```

#### 3.5.2 分组

侧边栏中所有分组（自定义组 + 默认组）默认展开：
```javascript
// 默认展开所有组
appState.groupsExpanded = {};  // 只记录已折叠的组
```

### 3.6 组拖拽排序

**位置**：侧边栏

- 组之间可拖拽调整顺序（当前仅卡片可拖拽）
- 拖拽组时，组内卡片跟随移动
- 新增"默认组"始终不可删除，但可拖拽调整位置（建议固定在底部）

`appState` 新增字段：
```javascript

  // ... 现有字段
  recitationGroups: [],              // 背诵组列表

    dailyWordQuota: 500,
    dailyQuotaLevel: 'B',
    hintCount: 2                     // ★ 新增：提示显示数量
  }
}
```

`cardStates[title]` 新增字段：
```javascript
cardStates[title] = {
  // ... 现有字段
  myCreation: '',                    // 创作内容
  replaceAnswer: false,              // 替代开关
  editedKeywords: null               // 用户编辑后的关键词（null=使用原始）
}
```
