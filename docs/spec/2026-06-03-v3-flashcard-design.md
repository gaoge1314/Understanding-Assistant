# 第三次更新 — 背诵模式 + 概念填空 + 三视图导航

## 概述

在现有交互式 HTML 学习应用基础上增加三大核心功能：

1. **三视图导航** — 学习/背诵/复习三个视图通过顶部Tab切换
2. **背诵模式 + 记忆曲线** — 翻转卡 + SM-2 间隔重复 + 每日字数配额
3. **概念填空** — 右侧栏第二种重构交互模式，Agent 预生成空白数据

## 1. 三视图导航

### 1.1 方案

顶部导航栏中增加三个 Tab 按钮，替代当前的"交互模式/背诵模式"切换开关：

```
┌──────────────────────────────────────────────┐
│  📚 灾害学    [📖 学习] [🎴 背诵] [📅 复习]  │
└──────────────────────────────────────────────┘
```

- 当前激活的 Tab 高亮
- 点击切换视图，每个视图独立渲染布局
- 切换时保持卡片学习状态（localStorage 中的数据不变）

### 1.2 各视图布局

| 视图 | 布局 | 说明 |
|:---|:---|:---|
| **学习视图** | 三栏（左目录 / 主内容区 / 右侧栏） | 与现有完全一致，右侧栏新增概念填空模式 |
| **背诵视图** | 翻转卡居中（无侧栏） | 进入时先弹每日字数设置面板，确认后翻转背诵 |
| **复习视图** | 队列列表 + 进度卡片 | 显示记忆曲线概况、复习历史（轻量实现） |

### 1.3 复习视图（轻量）

只显示：
- 每日字数档次及当前进度（今日已背 N 字 / 配额 M 字）
- 各卡片复习状态：未学 / 待复习（N 天后续） / 已掌握
- 点击卡片可跳转到学习视图查看详情

## 2. 背诵模式 + 记忆曲线

### 2.1 用户流程

```
① 切换背诵Tab → ② 弹出字数设置面板
   → ③ 系统计算推荐 → ④ 用户确认 → ⑤ 开始翻转背诵
```

### 2.2 每日字数设置

四个档次（单选）：

| 档次 | 每日字数 |
|:---|:---|
| A | 200 字 |
| B | 500 字 |
| C | 1000 字 |
| D | 3000 字 |

- 字数只统计每张卡片 `answer` 字段（考试默写内容）
- 首次使用弹出设置面板，后续可随时在复习视图或设置中修改
- 设置保存在 `localStorage.userSettings.dailyWordQuota`

### 2.3 推荐算法

系统按以下优先级选卡片，直到总字数满足要求：

```
优先级 1: 待复习卡片
  条件: reviewRecord[nextReviewDate] <= today
  排序: 按 nextReviewDate 升序（最早的先复习）

优先级 2: 关联卡片
  条件: 未学 (level=0) 且与待复习卡片有 knowledge_interfaces 关联
  排序: 按关联数降序（关联最多的优先）

优先级 3: 顺序补充
  条件: 未学 (level=0)
  排序: 按 cardOrder 顺序
```

**字数约束：**
- 总字数 ≥ `dailyWordQuota × 90%`
- 总字数 ≤ `dailyWordQuota × 108%`
- 超出上限 → 移除最后一张卡片再检查
- 低于下限 → 提示"今日推荐不足配额90%，是否增加新卡片？"（若用户拒绝，则按当前推荐结果开始背诵）

### 2.4 记忆曲线（SM-2 改良版）

每张卡片维护复习状态：

```javascript
reviewRecord: {
  "<卡片标题>": {
    level: 0,              // 0=未学, 1~6=复习级别
    nextReviewDate: "",    // 下次复习日期 YYYY-MM-DD
    interval: 1,           // 当前间隔天数
    reviewCount: 0,        // 已复习次数
    lastReviewDate: null   // 上次复习日期
  }
}
```

**间隔计划（level → interval）：**

| Level | 间隔 | 累计 |
|:---|:---|:---|
| 0 (未学) | — | — |
| 1 | 1 天 | 1 天 |
| 2 | 3 天 | 4 天 |
| 3 | 7 天 | 11 天 |
| 4 | 21 天 | 32 天 |
| 5 | 60 天 | 92 天 |
| 6 | 180 天 | 272 天 |

**反馈规则：**
- **记得** → `level += 1`，`interval = 对应天数`，更新 `nextReviewDate`
- **忘了** → `level = 1`，`interval = 1`，`nextReviewDate = 明天`

### 2.5 翻转交互

- **正面**：卡片标题 + 进度标签（"第 3/5 张"）
- **背面**：`answer` 字段内容
- **操作**：点击卡片翻转（CSS 3D Transform）、"记得 👍" 按钮、"忘了 👎" 按钮
- **进度**：底部显示当前第几张 / 今日共几张

## 3. 概念填空

### 3.1 数据扩展

`reconstruct_type` 改为数组：

```json
{
  "reconstruct_type": ["reorder", "fill-blank"],
  "title": "灾害系统四要素",
  "answer": "灾害系统由孕灾环境、致灾因子、承灾体和灾情四要素构成...",
  "blanks": [
    {
      "sentence": "灾害系统由____、致灾因子、____和灾情四要素构成。",
      "answers": ["孕灾环境", "承灾体"],
      "hint": "填写另外两个要素的名称"
    },
    {
      "sentence": "____是灾害发生的背景条件，____是灾害的直接触发因素。",
      "answers": ["孕灾环境", "致灾因子"],
      "hint": "填写对应的要素名称"
    }
  ]
}
```

- `reconstruct_type` 为空数组 `[]` 表示不适合任何重构模式
- `blanks` 字段由 `generate_app.py` 在生成 data.json 时预生成

### 3.2 生成规则

`generate_app.py` 对每张卡片执行：

1. 从 `answer` 字段中找到**技术术语**（优先级：`decomposition` 中的条目 > `memory_techniques.keywords` 中的关键词 > 长度 ≥3 字的名词性短语）
2. 对每个术语，在原句中用 `____` 替换。一个句子最多挖 2 个空
3. 生成 2-3 道挖空题
4. 写入 `blanks` 数组
5. `reconstruct_type` 添加 `"fill-blank"`

### 3.3 交互

- 右侧栏顶部增加**下拉选择框**，选项为 `["概念重组", "概念填空"]`
- 下拉框仅显示当前卡片 `reconstruct_type` 中包含的模式
- 概念填空交互：
  - 显示第一道填空句子，`____` 处为输入框
  - 用户填写后点击"提交"
  - 比对答案：正确标记绿色，错误标记红色+显示正确答案
  - 点击"下一题"进入下一道空白
  - 所有题目完成后显示得分

## 7. 技术要点

- **翻转动画**：CSS `transform: rotateY(180deg)` + `backface-visibility: hidden`
- **Tab切换**：先隐藏所有视图，再显示选中视图，切换时 retain 各视图独立状态
- **字数计算**：`answer.length`，去除空白字符后计算
- **推荐算法边界**：如果待复习卡片+关联卡片仍不足 90%，提示用户手动确认是否继续
- **记忆曲线初始化**：用户首次完成背诵（记或忘）时初始化 `reviewRecord`，未背诵的卡片不占队列
